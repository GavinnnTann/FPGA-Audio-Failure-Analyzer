# CMOD A7 Recorder Project Reference

## Scope
This document is the single reference for the CMOD A7 real-time acoustic monitoring project based on recorder_top.

Target board: Digilent CMOD A7 (xc7a35tcpg236-1)
Primary function: capture microphone audio via I2S, compute STFT spectrogram and RMS amplitude, and stream telemetry + spectrogram slices to ESP32 display over UART.

## Build Target Summary
- Top module: recorder_top
- Source root used by build: src_main/
- Constraint file used by build: constraints/recorder.xdc
- Build scripts: scripts/build.ps1 and scripts/build.tcl
- IP repository: ip/ (contains xfft_1.xci for Xilinx FFT v9.1)

## Verilog Files Related to recorder_top
All recorder_top-related Verilog files are located in src_main.

### Audio Capture & Decimation
1. recorder_top.v
- Role: project top-level integration.
- Main functions:
  - Consumes microphone samples from i2s_receiver.
  - Instantiates FFT frontend pipeline.
  - Routes decimated samples to FFT.
  - Outputs spectrogram frame data.
  - Frames UART packet: AA 55 result rms flags seq metric checksum.
  - Drives status LEDs.

2. i2s_receiver.v
- Role: INMP441 interface block.
- Main functions:
  - Generates/uses I2S timing.
  - Captures 24-bit sample stream.
  - Asserts sample_valid for recorder_top processing.

3. uart_tx.v
- Role: UART byte transmitter.
- Main functions:
  - 8N1 serialization.
  - Parameterized baud generation.
  - Handshake signals used by recorder_top frame FSM.

### FFT Pipeline
4. fft_frontend.v
- Role: Top-level FFT integration module.
- Main functions:
  - Orchestrates window buffering → FFT computation → magnitude extraction → feature quantization.
  - Inputs: raw I2S samples (16-bit, 46.875 kHz, sign-extended to 24-bit), sample_valid strobe.
  - Outputs: `spec_bin_magnitude[1023:0]`, `spec_bin_feature8[511:0]`, `spec_frame_valid`, `spec_frame_index`.
  - Frame rate: ~20 Hz (65 ms per spectrogram).
  - Delayed bin index (`mag_bin_index_d1`) ensures feature8 storage writes are aligned with quantizer pipeline.
  - `spec_frame_valid` derived from `feature_line_count` (counts feature8_valid events, not mag_valid), ensuring all 64 feature bins are committed before frame pulse.

5. fft_window_buffer.v
- Role: Hann window application and sample buffering.
- Main functions:
  - Accumulates 512 decimated samples into a ring buffer.
  - Multiplies each sample by a Hann window coefficient (Q1.15 fixed-point).
  - DSP-pipelined multiply: registered inputs (`mul_a`, `mul_b`) and product (`win_mult`) enable Vivado DSP48 MREG/PREG inference.
  - Streams windowed samples to FFT core with backpressure handling (fft_ready).
  - ROM: hann_512_q15.mem (512 × 16-bit coefficients).

6. fft_magnitude.v
- Role: Complex-to-magnitude conversion and bin downsampling.
- Main functions:
  - Extracts I/Q components from FFT output (80-bit bus).
  - Computes magnitude: `mag ≈ 0.96×max(|I|,|Q|) + 0.398×min(|I|,|Q|)` (Alphamax+Betamax).
  - Downsamples first 256 FFT bins → 64 output bins (every 4th bin; bins 256–511 are conjugate mirrors and are skipped).
  - Outputs: 16-bit magnitude + 6-bit bin index per cycle.
  - Frequency coverage: 0 to ~23.4 kHz (Nyquist), bin spacing ≈ 366 Hz.

7. fft_feature_quantizer.v
- Role: 16-bit magnitude to 8-bit compressed feature conversion.
- Main functions:
  - Leading-one position detection for log2 integer part.
  - 4-bit fractional interpolation from normalized mantissa.
  - Output format: `{lz[3:0], norm[14:11]}` — 16 log-scale bands × 16 fractional steps = monotonic 0–255 range.
  - Outputs 8-bit companded feature per magnitude sample (1-cycle pipeline).

8. spectrogram_buffer_64x64.v
- Role: 64x64x8 BRAM staging buffer.
- Main functions:
  - Writes one 64-bin feature line per `line_valid` strobe.
  - Signals `frame_complete` when line_index reaches 63.
  - Provides `latest_line` passthrough for downstream consumers.

9. xfft_1 (IP Core)
- Role: Xilinx FFT v9.1 IP instantiation.
- Configuration:
  - Transform size: 512-point.
  - Input: 48-bit AXI-Stream (24-bit real, sign-extended).
  - Output: 80-bit packed (I and Q, 34-bit each, signed).
  - Architecture: Pipelined streaming I/O.
  - Throttle scheme: Real-time (no backpressure blocking).
  - Output ordering: Natural (bins 0–511 in order).

### Memory Files
10. hann_512_q15.mem
- Role: Hann window coefficients for fft_window_buffer.v.
- Format: 512 lines, hex, Q15 fixed-point (range: 0x0000–0x7FFF).
- Generated by: gen_hann_coeffs.py.

### RMS Telemetry Path
The original amplitude-only features (RMS, flags) remain active in recorder_top. The UART FSM sends an 8-byte RMS frame followed by a 6-byte spectrogram slice packet each cycle.

## Communication Protocol (FPGA -> ESP32)

### Primary Telemetry Frame (Original 8-byte format, still active)
Fixed 8-byte frame format:
1. 0xAA (sync)
2. 0x55 (sync)
3. result
4. rms
5. flags
6. seq
7. metric (placeholder for CNN confidence or spectrogram peak bin)
8. checksum = AA XOR 55 XOR result XOR rms XOR flags XOR seq XOR metric

### Spectrogram Slice Packets (Burst Mode)
After each RMS frame, the FPGA burst-transmits all 64 spectrogram slice packets:
- Sync: `0xDD 0x77`
- Format: `[0xDD] [0x77] [bin_index: 0–63] [bin_low] [bin_high] [checksum]`
- checksum = 0xDD XOR 0x77 XOR bin_index XOR bin_low XOR bin_high
- All 64 bins burst-transmitted after each RMS frame (~3.8 ms burst, ~18 Hz refresh)
- Magnitude is 16-bit linear from `spec_bin_magnitude`, little-endian
- Each bin value latched into `latched_spec_bin` at packet start for data integrity

See UART_SPECTROGRAM_PROTOCOL.md for full protocol details.

**8-bit feature path** (available but not yet transmitted over UART):
- `spec_bin_feature8[511:0]`: 64 bins × 8-bit log-compressed features (`{lz[3:0], norm[14:11]}`, monotonic 0–255)
- Written using delayed bin index (`mag_bin_index_d1`) for pipeline-aligned storage
- Staged in `spectrogram_buffer_64x64` for future CNN input

Typical UART settings:
- 1,000,000 baud
- 8 data bits, no parity, 1 stop bit

## Build and Programming Flows
From workspace root:

- Build bitstream:
  powershell -ExecutionPolicy Bypass -File scripts/build.ps1 -Action build

- Program volatile FPGA SRAM (clears on power cycle):
  powershell -ExecutionPolicy Bypass -File scripts/build.ps1 -Action program

- Flash non-volatile configuration memory (boots after power cycle):
  powershell -ExecutionPolicy Bypass -File scripts/build.ps1 -Action flash

- Build and flash in one step:
  powershell -ExecutionPolicy Bypass -File scripts/build.ps1 -Action allflash

## From-Scratch Onboarding (Day 1)
1. Install Vivado ML Edition and confirm Artix-7 device support is present.
2. Connect CMOD A7 over USB-JTAG.
3. Wire INMP441 to PMOD JA using the exact mapping in this document.
4. Wire UART:
  - CMOD `uart_tx` on package pin N3 (edge Pin 18) -> ESP32 RX.
  - Connect board grounds together.
5. Build and program:
  - Build: `powershell -ExecutionPolicy Bypass -File scripts/build.ps1 -Action build`
  - Program: `powershell -ExecutionPolicy Bypass -File scripts/build.ps1 -Action program`
6. Verify serial protocol on ESP32 side:
  - UART 1,000,000 baud, 8N1.
  - Frame length 8 bytes.
  - Sync bytes `AA 55`.
  - Checksum includes flags, seq, and metric bytes.
7. If persistence after power cycle is required, run:
  - `powershell -ExecutionPolicy Bypass -File scripts/build.ps1 -Action flash`

## Simulation and Verification

### FFT Testbench
A simplified testbench is available at `testbench/fft_test_simple.v` for pipeline validation without full IP simulation:

**To run:**
```powershell
.\scripts\run_sim.ps1 -SimTime 100
```

**Output:**
- VCD waveform: `fft_sim/fft_test_simple.vcd` (open in Surfer, GTKWave, etc.)
- Signals monitored: `phase_accum`, `sample_ena`, `sample_count`, `frame_count`, `sine_sample`
- Expected behavior: sample counter increments with frame counter every 512 samples

**Verification checklist:**
- [ ] `sample_ena` pulses regularly (~every 128 ns @ 12 MHz ÷ 6)
- [ ] `sample_count` increments 0→1→2→... at each pulse
- [ ] `frame_count` increments after 512 samples
- [ ] No timing violations in synthesis report

### FFT Backpressure Testbench
`testbench/fft_backpressure_tb.v` validates the window buffer under AXI-Stream backpressure conditions:

**Purpose**: Regression test for no-drop behavior when FFT consumer stalls.

**3-Phase Test:**
1. Fill ring buffer (512 samples, no stalls, verifies initial stream)
2. Backpressure test (HOP_N × 3 samples with random fft_ready deassertion every 7th sample)
3. Extended run (2048 samples with periodic stalls every 32 samples)

**Validation:**
- Frame continuity: each completed frame contains exactly FFT_N samples
- Reports total samples fed, streamed samples, completed frames, stall cycles, and error count
- RESULT: PASS/FAIL printed at end

**To run** (Vivado simulation):
```powershell
.\scripts\build.ps1 -Action simulate
```

### What Is Implemented Now (Operational Snapshot)
- Captures INMP441 microphone samples through I2S (46.875 kHz).
- Decimates audio stream by 6 to ~7.8125 kHz for RMS amplitude path.
- Feeds raw 46.875 kHz samples (sign-extended 16→24 bit) to FFT path for full bandwidth.
- Applies 512-point Hann windowing with hop=64 overlap scheduler (DSP-pipelined multiply path).
- Computes real FFT via Xilinx FFT v9.1 IP core.
- Extracts magnitude from I/Q using Alphamax+Betamax and downsamples first 256 bins (every 4th) to 64 output bins (0–23.4 kHz).
- Quantizes magnitudes to 8-bit log-compressed features: `{lz[3:0], norm[14:11]}` = monotonic 0–255 range.
- Feature8 storage uses delayed bin index (`mag_bin_index_d1`) for pipeline-aligned writes.
- Frame-valid pulse (`spec_frame_valid`) derived from feature8_valid count, ensuring all 64 bins are committed.
- Stages features in 64x64x8 spectrogram buffer.
- Outputs packed spectrogram (1024-bit, 64 × 16-bit bins + 512-bit, 64 × 8-bit features) with frame-valid pulse.
- Computes windowed mean-absolute amplitude (RMS path, for backward compatibility).
- Sends UART burst to ESP32: 8-byte RMS telemetry + 64 × 6-byte spectrogram slices (~18 Hz full refresh).
- Dual UART outputs: uart_tx (N3, ESP32) and uart_tx_usb (J18, onboard FTDI USB bridge).

### UART Protocol Notes (Implemented)
- Baud: 1,000,000
- Format: 8N1
- RMS frame: AA 55 result rms flags seq metric checksum (8 bytes)
- Spectrogram slice: DD 77 bin_idx bin_lo bin_hi checksum (6 bytes)
- RMS checksum: AA XOR 55 XOR result XOR rms XOR flags XOR seq XOR metric
- Spec checksum: DD XOR 77 XOR bin_idx XOR bin_lo XOR bin_hi

Additional protocol notes:
- flags[0] = 1 indicates FPGA active.
- metric is currently a placeholder byte from FPGA (set to 0 until CNN confidence is wired).
- seq increments after each complete RMS+spec cycle.
- Frame cadence depends on WINDOW_SAMPLES in src_main/recorder_top.v (~50 ms).
- All 64 spectrogram bins burst-transmitted after each RMS frame (~3.8 ms burst, ~18 Hz full refresh rate).

### Quick Troubleshooting
- No UART data on ESP32:
  - Verify physical wire is on N3 edge Pin 18.
  - Verify shared ground between boards.
  - Verify ESP32 UART baud is 1,000,000.

- Sync bytes not detected:
  - Confirm parser expects 8-byte frames, not legacy 5/6/7-byte frames.
  - Check checksum logic includes flags, seq, and metric bytes.

- Works until power cycle:
  - Use -Action flash (non-volatile), not only -Action program.
